## 一、需求说明

实现**成语接龙多轮对话**功能:
1. 用户说"开始成语接龙"启动游戏
2. AI 先出一个成语(第一次提示词:"本次开始成语接龙游戏,你先出一个词。第一个词难度别太高了")
3. 进入聆听模式,用户接词
4. AI 根据用户说的成语继续接词(后续提示词:"用户说了xxx成语,你接着说一次词")
5. 如果用户 30 秒不说话,自动退出游戏
6. 用户说"退出/结束/停止"可手动退出
7. 游戏全程**无需唤醒词**,AI 说完自动聆听

## 二、核心实现

### 2.1 修改文件

**文件**: `app/src/main/java/com/phicomm/speaker/device/custom/handler/PhicommChatHandler.java`

### 2.2 新增状态变量

```java
// 成语接龙游戏状态
private boolean isIdiomGameMode = false;  // 是否在成语接龙模式
private int idiomGameTurn = 0;  // 当前轮数
private String lastUserIdiom = null;  // 用户说的最后一个成语
private static final int MAX_IDLE_TIMEOUT = 30000;  // 30秒超时退出

// 超时处理器
private Handler timeoutHandler = new Handler(Looper.getMainLooper());
private Runnable timeoutRunnable;
```

### 2.3 关键修改点

#### 1. acceptInboundEvent0: 优先级提升

```java
@Override
public boolean acceptInboundEvent0(NLU evt) {
    String text = evt.getText();
    
    // 成语接龙模式下,不拦截任何输入(全部消费) ✅
    if (isIdiomGameMode) {
        LogMgr.d(TAG, "成语接龙模式,拦截所有输入: " + text);
        return true;
    }
    
    // 检查是否是成语接龙启动指令(最高优先级) ✅
    if (text.contains("成语接龙") || text.contains("开始成语") || text.contains("成语游戏")) {
        LogMgr.d(TAG, "检测到成语接龙启动指令: " + text);
        return true;
    }
    
    // 原有逻辑:音乐/天气放行
    // ...
}
```

**优先级**: 成语接龙 > 音乐/天气 > 普通对话

#### 2. eventReceived: 游戏逻辑

```java
@Override
public void eventReceived(final NLU evt, final ANTHandlerContext ctx) throws Exception {
    final String userInput = evt.getText();
    
    // 取消超时定时器(用户说话了)
    if (isIdiomGameMode) {
        timeoutHandler.removeCallbacks(timeoutRunnable);
    }
    
    ThreadUtils.execute(new Runnable() {
        @Override
        public void run() {
            String response;
            
            // 情况1: 启动成语接龙游戏
            if (!isIdiomGameMode && (userInput.contains("成语接龙") || ...)) {
                isIdiomGameMode = true;
                idiomGameTurn = 0;
                lastUserIdom = null;
                
                // 第一次提示词
                String prompt = "本次开始成语接龙游戏,你先出一个成语。" +
                               "要求:\n" +
                               "1. 第一个成语难度别太高,用常见的成语\n" +
                               "2. 只回复成语本身,不要解释,不要其他内容\n" +
                               "3. 必须是四字成语";
                response = openAIClient.chat(prompt);
                response = "好的,我们开始成语接龙!我先开始:" + response;
                
            // 情况2: 游戏中用户接词
            } else if (isIdiomGameMode) {
                // 检查退出指令
                if (userInput.contains("退出") || userInput.contains("结束") || ...) {
                    isIdiomGameMode = false;
                    response = "好的,成语接龙游戏结束!你一共完成了 " + idiomGameTurn + " 轮对话,非常棒!";
                    ctx.playTTS(response);
                    return;  // 播报后退出
                }
                
                // 正常接词
                idiomGameTurn++;
                lastUserIdom = userInput;
                
                // 后续提示词(包含用户说的词)
                String prompt = "成语接龙游戏中。\n" +
                               "用户说了:「" + userInput + "」\n" +
                               "请你根据成语接龙规则接一个成语。\n" +
                               "要求:\n" +
                               "1. 只回复成语本身,不要解释\n" +
                               "2. 必须是四字成语\n" +
                               "3. 不能重复使用已经说过的成语";
                response = openAIClient.chat(prompt);
                
            // 情况3: 普通对话
            } else {
                response = openAIClient.chat(userInput);
            }
            
            // TTS 播报
            ctx.playTTS(response);
        }
    });
}
```

#### 3. onTTSEventPlayingEnd: 自动聆听

```java
@Override
public boolean onTTSEventPlayingEnd(ANTHandlerContext ctx) {
    if (isIdiomGameMode) {
        // 成语接龙模式:直接进入 ASR,继续聆听 ✅
        LogMgr.d(TAG, "成语接龙模式:进入 ASR 继续聆听");
        reset();
        
        // 启动超时定时器(30秒)
        timeoutRunnable = new Runnable() {
            @Override
            public void run() {
                if (isIdiomGameMode) {
                    LogMgr.d(TAG, "成语接龙超时,自动退出");
                    isIdiomGameMode = false;
                    ctx.playTTS("超时未检测到语音,成语接龙游戏结束");
                    ctx.enterWakeup(false);
                }
            }
        };
        timeoutHandler.postDelayed(timeoutRunnable, MAX_IDLE_TIMEOUT);
        
        // 延迟 500ms 再进入 ASR,确保 TTS 完全停止
        new Handler(Looper.getMainLooper()).postDelayed(new Runnable() {
            @Override
            public void run() {
                ctx.enterASR();  // ← 无需唤醒词,直接聆听!
            }
        }, 500);
        
        return true;
    } else {
        // 普通模式:回到唤醒状态
        exit();
        return true;
    }
}
```

#### 4. doInterrupt: 中断处理

```java
@Override
public void doInterrupt(ANTHandlerContext ctx, String interruptType) {
    if (eventReceived) {
        ctx.cancelTTS();
        
        // 取消超时定时器
        timeoutHandler.removeCallbacks(timeoutRunnable);
        
        // 中断时退出成语接龙模式
        isIdiomGameMode = false;
        idiomGameTurn = 0;
        lastUserIdom = null;
        
        reset();
    }
}
```

## 五、关键技术点

### 5.1 优先级控制

成语接龙模式具有**最高优先级**:

```java
// 成语接龙模式下,不拦截任何输入(全部消费)
if (isIdiomGameMode) {
    return true;  // ← 所有输入都走大模型
}
```

这确保了:
- ✅ 游戏过程中,即使是"播放音乐"也会被当成游戏输入
- ✅ 游戏优先级 > 音乐/天气 > 普通对话

### 5.2 多轮对话机制

通过 `ctx.enterASR()` 实现自动聆听:

```java
// TTS 结束后
if (isIdiomGameMode) {
    reset();
    
    // 延迟 500ms(确保 TTS 停止)
    new Handler(Looper.getMainLooper()).postDelayed(() -> {
        ctx.enterASR();  // ← 直接进入识别,无需唤醒词!
    }, 500);
}
```

### 5.3 超时退出机制

防止设备一直等待:

```java
// 启动超时定时器
timeoutRunnable = new Runnable() {
    @Override
    public void run() {
        if (isIdiomGameMode) {
            isIdiomGameMode = false;
            ctx.playTTS("超时未检测到语音,成语接龙游戏结束");
            ctx.enterWakeup(false);
        }
    }

```

**用户说话时取消定时器**:

```java

    timeoutHandler.removeCallbacks(timeoutRunnable);  // ← 取消超时
}
```

### 5.4 上下文记忆

通过 `lastUserIdom` 记录用户说的最后一个成语:

## 九、注意事项

### 9.1 延迟进入 ASR

在 TTS 结束后延迟 500ms 再进入 ASR:

```java
new Handler(Looper.getMainLooper()).postDelayed(() -> {
    ctx.enterASR();
}, 500);
```

**原因**: 确保 TTS 完全停止,避免录到 TTS 尾音

### 9.2 超时定时器管理

- 用户说话时**立即取消**超时定时器
- TTS 结束后**重新启动**超时定时器
- 退出游戏时**清理**超时定时器

### 9.3 状态重置

在以下情况重置游戏状态:
- 用户手动退出
- 超时自动退出
- 被打断(doInterrupt)
- 网络异常

### 9.4 错误处理

- AI 返回空回复: "我想想...这个有点难,你赢了!"
- 网络异常: "网络异常,游戏结束"
- 超时: "超时未检测到语音,成语接龙游戏结束"
