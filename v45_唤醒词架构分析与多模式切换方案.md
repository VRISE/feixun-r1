## 一、唤醒词工作原理深度解析

### 1.1 整体架构

R1 设备使用 **云知声 USC 引擎** + **Netty Pipeline 事件驱动架构** 处理唤醒词:

```
用户说话 "小孩同学"
    ↓
[USC 语音识别引擎]
    ├─ 本地 JSGF 词槽匹配
    ├─ 声学模型打分
    └─ 输出: LocalASR 对象
    ↓
[NativeANTEngine] (事件触发器)
    └─ fireASRResult(3201, result_json)  ← 事件类型 3201 = WAKEUP_EVENT_RECOGNITION_SUCCESS
    ↓
[ANTPipeline] (Netty 管道)
    └─ 依次经过所有 Handler 的 onWakeupResult()
    ↓
[NLUDispatcher] (核心路由器)
    ├─ 判断唤醒词类型(主唤醒词/功能唤醒词/竞品词)
    ├─ 触发 LocalASR 事件
    └─ 进入语音识别会话
    ↓
[Session Handlers] (业务处理器)
    ├─ DefaultChatHandler (聊天)
    ├─ DefaultMusicHandler (音乐)
    ├─ DefaultSettingHandler (设置)
    └─ PhicommXXXHandler (自定义业务)
```

#### 步骤 3: NLUDispatcher 路由决策

**文件**: `NLUDispatcher.java` (⭐核心路由逻辑)

```java
@Override
protected boolean onWakeupResult(ANTHandlerContext ctx, String result) {
    LogMgr.d("NLUDispatcher", "onWakeupResult:" + result);
    
    // 1. 解析唤醒词
    LocalASR localASR = this.c.from(result).getLocalASRList().get(0);
    String trim = localASR.getRecognitionResult().trim();  // "小孩同学"
    
    // 2. 判断是否为竞品词(如"天猫精灵")
    if (a(ctx.androidContext(), trim)) {
        LogMgr.d("NLUDispatcher", trim + " is CompetitionWord, return");
        return true;  // 拦截,不响应
    }
    
    // 3. 判断是否为功能唤醒词(如"上一首""下一首")
    else if (b(ctx.androidContext(), trim) && a(localASR.getScore())) {
        // b() = 不是主唤醒词
        // a() = 分数超过功能唤醒阈值
        LogMgr.d("NLUDispatcher", "isFunctionWakeupWord:" + trim);
        return a(ctx, localASR);  // 触发功能词事件
    }
    
    // 4. 判断是否为主唤醒词(如"小孩同学""小讯小讯")
    else if (c(ctx.androidContext(), trim) && b(localASR.getScore())) {
        // c() = 是主唤醒词
        // b() = 分数超过主唤醒阈值
        LogMgr.d("NLUDispatcher", "isMainWakeupWord:" + trim);
        return false;  // 放行,进入正常语音识别流程
    }
    
    return true;  // 默认拦截
}
```

**判断逻辑详解**:

| 方法 | 判断条件 | 代码位置 |
|------|---------|---------|
| `a(context, word)` | 是否为竞品词 | `UserPerferenceUtil.getCmopetitionWord(context).contains(word)` |
| `b(context, word)` | 是否**不是**主唤醒词 | `!UserPerferenceUtil.getMainWakeupWord(context).contains(word)` |
| `c(context, word)` | 是否**是**主唤醒词 | `UserPerferenceUtil.getMainWakeupWord(context).contains(word)` |
| `a(score)` | 分数是否超过功能唤醒阈值 | `score > ANTConfigPreference.FUNCTION_WAKEUP_BENCHMARK` |
| `b(score)` | 分数是否超过主唤醒阈值 | `score > ANTConfigPreference.effectWakeupBenchmark` |

#### 步骤 4: 进入会话处理

当主唤醒词被识别后,系统会:

1. **停止唤醒监听**: `ctx.enterWakeup(false)`
2. **开始语音识别**: 等待用户说出指令
3. **NLU 意图识别**: 将语音转为结构化意图
4. **Pipeline 路由到 Handler**: 根据 intent.service 分发

## 二、唤醒词分类体系

### 2.1 三类唤醒词

| 类型 | 示例 | 阈值 | 行为 | 配置位置 |
|------|------|------|------|---------|
| **主唤醒词** | 小讯小讯、小孩同学、交接手续 | `effectWakeupBenchmark` | 进入语音对话模式 | `arrays.xml` + SharedPreferences |
| **功能唤醒词** | 上一首、下一首、暂停、播放 | `FUNCTION_WAKEUP_BENCHMARK` | 直接执行功能,不进入对话 | `arrays.xml` |
| **竞品词** | 天猫精灵、小爱同学 | 无 | 忽略,不响应 | `UserPerferenceUtil.getCmopetitionWord()` |

```java
    arrayList.addAll(hashSet);
    return arrayList;
}
```

## 三、当前架构的局限性

### 3.1 问题: 所有唤醒词行为一致

**现状**:
```
"小孩同学" → 进入对话模式 → 等待用户指令 → NLU 识别 → 执行
"小讯小讯" → 进入对话模式 → 等待用户指令 → NLU 识别 → 执行
"交接手续" → 进入对话模式 → 等待用户指令 → NLU 识别 → 执行
```

**用户期望**:
```
"捣蛋鬼"   → 直接进入"捣蛋模式"(幽默对话风格)
"英语陪练师" → 直接进入"英语对话模式"
"成语接龙游戏伙伴" → 直接进入"成语接龙游戏"
```

### 3.2 核心瓶颈

**NLUDispatcher.onWakeupResult() 只判断"是不是主唤醒词",不区分"是哪个唤醒词"**:

```java
// 当前逻辑
if (c(ctx.androidContext(), trim)) {  // 只要是主唤醒词
    return false;  // 全部放行,进入相同流程
}

// 缺失的逻辑
if (trim.equals("捣蛋鬼")) {
    // 进入捣蛋模式
} else if (trim.equals("英语陪练师")) {
    // 进入英语模式
}
```

## 四、多模式唤醒词架构设计方案

### 4.1 设计目标

✅ **呼唤不同唤醒词 → 自动进入不同人格/模式**  
✅ **无需说出第一次指令**  
✅ **保留原有唤醒词兼容性**  
✅ **支持动态切换模式**

### 4.2 架构方案

```
用户说 "捣蛋鬼"
    ↓
[USC 引擎识别] → LocalASR.recognition_result = "捣蛋鬼"
    ↓
[NLUDispatcher.onWakeupResult()]
    ├─ 判断: 是主唤醒词 → return false (放行)
    └─ 新增: 发布 WakeupModeEvent
    ↓
[Pipeline.fireUserEventTriggered(new WakeupModeEvent("捣蛋鬼"))]
    ↓
[ModeRouterHandler] (新增) ⭐核心路由器
    ├─ 读取唤醒词 → "捣蛋鬼"
    ├─ 查询映射表 → ModeConfig.MISCHIEVOUS_MODE
    ├─ 设置全局模式: ContextManager.setCurrentMode("捣蛋鬼")
    ├─ 播放模式切换提示音: "捣蛋鬼模式已激活,嘿嘿嘿~"
    └─ 触发模式初始化 NLU
    ↓
[Session Handler 根据模式路由]
    ├─ PhicommChatHandler (优先级 315)
    │   └─ 检查: ContextManager.getCurrentMode()
    │       ├─ "捣蛋鬼" → 使用捣蛋鬼系统提示词调用大模型
    │       ├─ "英语陪练师" → 使用英语老师提示词
    │       └─ "成语接龙" → 启动成语接龙游戏逻辑
    └─ 其他 Handler...
```

### 4.3 核心实现

#### 实现 1: 扩展 NLUDispatcher 发布唤醒词事件

**文件**: `NLUDispatcher.java`

```java
@Override
protected boolean onWakeupResult(ANTHandlerContext ctx, String result) {
    LogMgr.d("NLUDispatcher", "onWakeupResult:" + result);
    LocalASR localASR = this.c.from(result).getLocalASRList().get(0);
    String trim = localASR.getRecognitionResult().trim();
    
    // 原有逻辑:判断唤醒词类型
    if (a(ctx.androidContext(), trim)) {
        return true;  // 竞品词,拦截
    } else if (b(ctx.androidContext(), trim) && a(localASR.getScore())) {
        return a(ctx, localASR);  // 功能词
    } else if (c(ctx.androidContext(), trim) && b(localASR.getScore())) {
        
        // ⭐ 新增: 发布唤醒词模式切换事件
        ctx.pipeline().fireUserEventTriggered(new WakeupModeEvent(trim));
        
        return false;  // 放行,进入语音识别
    }
    
    return true;
}
```

```java
package com.phicomm.speaker.device.custom.handler;

import com.phicomm.speaker.device.custom.context.ContextManager;
import com.phicomm.speaker.device.custom.event.WakeupModeEvent;
import com.phicomm.speaker.device.custom.mode.ModeConfig;
import com.unisound.vui.engine.ANTHandlerContext;
import com.unisound.vui.handler.SimpleUserEventInboundHandler;
import com.unisound.vui.util.LogMgr;

/**
 * 唤醒词模式路由器
 * 监听 WakeupModeEvent,根据唤醒词切换系统模式
 */
public class ModeRouterHandler extends SimpleUserEventInboundHandler<WakeupModeEvent> {
    
    private static final String TAG = ModeRouterHandler.class.getSimpleName();
    
    @Override
    protected boolean acceptInboundEvent0(WakeupModeEvent evt) throws Exception {
        // 只处理唤醒词模式事件
        return evt != null && evt.getWakeupWord() != null;
    }
    
    @Override
    protected void eventReceived(WakeupModeEvent evt, ANTHandlerContext ctx) throws Exception {
        String wakeupWord = evt.getWakeupWord();
        LogMgr.d(TAG, "Received wakeup mode event: " + wakeupWord);
        
        // 1. 查询唤醒词对应的模式配置
        ModeConfig modeConfig = ModeConfig.findByWakeupWord(wakeupWord);
        
        if (modeConfig == null) {
            LogMgr.d(TAG, "No mode configured for wakeup word: " + wakeupWord);
            return;
        }
        
        // 2. 切换当前模式
        ContextManager.setCurrentMode(modeConfig.getModeId());
        LogMgr.d(TAG, "Switched to mode: " + modeConfig.getModeName());
        
        // 3. 播放模式切换提示音
        String ttsTip = modeConfig.getEnterTip();
        if (ttsTip != null && !ttsTip.isEmpty()) {
            ctx.playTTS(ttsTip);
        }
        
        // 4. 触发模式初始化(可选)
        if (modeConfig.isAutoInit()) {
            ctx.pipeline().fireUserEventTriggered(modeConfig.getInitNLU());
        }
    }
    
    @Override
    protected void initPriority() {
        // 设置最高优先级,确保在业务 Handler 之前执行
        setPriority(400);
    }
}
```

```java
package com.phicomm.speaker.device.custom.context;

import com.phicomm.speaker.device.custom.mode.ModeConfig;

/**
 * 全局上下文管理器
 * 维护当前会话的模式状态
 */
public class ContextManager {
    
    private static String currentMode = ModeConfig.DEFAULT_MODE;
    private static long modeSwitchTime = 0;
    
    /**
     * 设置当前模式
     */
    public static void setCurrentMode(String modeId) {
        currentMode = modeId;
        modeSwitchTime = System.currentTimeMillis();
    }
    
    /**
     * 获取当前模式
     */
    public static String getCurrentMode() {
        return currentMode;
    }
    
    /**
     * 获取当前模式的配置
     */
    public static ModeConfig getCurrentModeConfig() {
        return ModeConfig.findByWakeupWord(getCurrentWakeupWord());
    }
    
    /**
     * 获取当前模式对应的唤醒词
     */
    public static String getCurrentWakeupWord() {
        // 反向查找
        for (Map.Entry<String, ModeConfig> entry : ModeConfig.WAKEUP_WORD_MAP.entrySet()) {
            if (entry.getValue().getModeId().equals(currentMode)) {
                return entry.getKey();
            }
        }
        return "小讯小讯";  // 默认
    }
    
    /**
     * 检查模式是否已过期(可选,比如30分钟无交互自动恢复默认)
     */
    public static boolean isModeExpired(long timeoutMs) {
        return System.currentTimeMillis() - modeSwitchTime > timeoutMs;
    }
    
    /**
     * 重置为默认模式
     */
    public static void resetToDefault() {
        currentMode = ModeConfig.DEFAULT_MODE;
    }
}
```

```java
@Override
protected void eventReceived(NLU evt, ANTHandlerContext ctx) throws Exception {
    super.eventReceived(evt, ctx);
    
    // ⭐ 获取当前模式
    String currentMode = ContextManager.getCurrentMode();
    ModeConfig modeConfig = ModeConfig.findByWakeupWord(
        ContextManager.getCurrentWakeupWord()
    );
    
    LogMgr.d(TAG, "Current mode: " + currentMode);
    
    // 根据不同模式调用不同的大模型逻辑
    if (ModeConfig.MISCHIEVOUS_MODE.equals(currentMode)) {
        // 捣蛋鬼模式
        handleMischievousMode(evt, ctx, modeConfig);
    } 
    else if (ModeConfig.ENGLISH_TUTOR_MODE.equals(currentMode)) {
        // 英语陪练师模式
        handleEnglishTutorMode(evt, ctx, modeConfig);
    }
    else if (ModeConfig.IDIOM_GAME_MODE.equals(currentMode)) {
        // 成语接龙模式
        handleIdiomGameMode(evt, ctx);
    }
    else {
        // 默认模式
        handleDefaultMode(evt, ctx);
    }
}

 * 英语陪练师模式处理
 */
private void handleEnglishTutorMode(NLU evt, ANTHandlerContext ctx, ModeConfig config) {
    String userInput = evt.getText();
    
    // 检查用户是否在用英语对话
    if (!isEnglish(userInput)) {
        ctx.playTTS("Try to speak in English, please! " + userInput);
        return;
    }
    
    // 调用英语老师大模型
    String systemPrompt = config.getSystemPrompt();
    String response = callLLMWithSystemPrompt(systemPrompt, userInput);
    
    ctx.playTTS(response);
}

/**
 * 成语接龙模式处理
 */
private void handleIdiomGameMode(NLU evt, ANTHandlerContext ctx) {
    // 成语接龙特殊逻辑(已在 v46 实现)
    String userInput = evt.getText();
    
    if (IdiomGameEngine.isValidIdiom(userInput)) {
        String nextIdiom = IdiomGameEngine.getNextIdiom(userInput);
        ctx.playTTS("好!我接: " + nextIdiom);
    } else {
        ctx.playTTS("这个不是成语哦,再想想!");
    }
}
```

## 十、总结

### 10.1 核心创新点

1. **唤醒词即模式**: 每个唤醒词不再只是"启动对话",而是"切换到特定人格/模式"
2. **零指令进入**: 呼唤唤醒词后直接进入模式,无需再说"进入XX模式"
3. **Pipeline 事件驱动**: 利用 Netty Pipeline 机制,解耦唤醒词识别和模式切换
4. **配置化设计**: 新增模式只需添加配置,无需修改核心逻辑

### 10.2 技术优势

- ✅ **向后兼容**: 不影响现有唤醒词和功能
- ✅ **高扩展性**: 新增模式只需添加配置
- ✅ **低耦合**: 通过事件机制解耦各模块
- ✅ **易维护**: 模式逻辑集中在 ModeConfig 和 PhicommChatHandler
