# 空闲插嘴模式 — 设计与实现文档

## 一、需求概述

### 系统架构

整个智能音响系统分为两层优先级：

| 优先级 | 模式 | 触发方式 | 说明 |
|--------|------|---------|------|
| **最高** | 命令模式 | JSGF 本地唤醒词语法 | 调高/调低音量、播放音乐等命令，多唤醒词呼唤 |
| **第二** | 插嘴模式 | 空闲期后台监听 | 空闲 30 秒后启动独立音频采集，检测到人声后 ASR → 大模型 → TTS |

### 核心需求

1. **命令模式优先级最高** — JSGF 本地语法匹配的命令（音量、播放等）永远优先进入，不受插嘴模式干扰
2. **插嘴模式仅在空闲期激活** — 所有其他模式（唤醒词/命令/TTS）退出 30 秒以上后才开始监听
3. **命中其他模式立即退出** — 只要唤醒词或命令被处理，插嘴模式立刻取消
4. **人声检测逻辑** — 持续监听人声，累计 5 秒以上语音 + 2 秒静音截断后送 ASR 识别
5. **无需 TTS 播放中插嘴** — TTS 播放说明有其他任务在运行，此时不进入插嘴模式

### 可行性结论

- 命令模式优先：系统已有 JSGF → ASR → NLU → Handler 链路，天然独立
- 空闲检测：`EavesdropperEngine` 已有 30 秒空闲定时器
- 音频采集：独立 `AudioRecord` 采集，不在 TTS 播放期间使用，避开 AEC 回声消除问题
- ASR 识别：`JniUscClient` 支持流式音频输入，可独立创建 ASR 会话
- 大模型回复：`EavesdropperHandler` + `OpenAIClient` 已实现完整链路

## 二、系统原有语音架构

```
[麦克风] → [AEC 回声消除] → [USC 引擎]
                              ├─ 唤醒词检测 (JSGF 本地语法, 始终监听)
                              └─ ASR 语音识别 (VAD 端点检测 + 云端识别)
                                             ↓
                                      [云端 NLU 服务]
                                             ↓
                              [ANT Engine Pipeline → Handler 链]
                                             ↓
                                      [TTS 离线合成 → 喇叭]
```

### 关键组件

| 组件 | 路径 | 说明 |
|------|------|------|
| VAD | `cn.yunzhisheng.asr.VAD` | JNI 封装的语音活动检测，仅在 ASR 模式下激活 |
| ASR 客户端 | `cn.yunzhisheng.asr.JniUscClient` | 流式音频输入：createNative → login → start → recognize → stop → getResult |
| ANT Pipeline | `com.unisound.vui.engine.ANTPipeline` | NLU 事件分发管道，Handler 链拦截消费 |
| EavesdropperEngine | `custom.engine.EavesdropperEngine` | 空闲定时器（已有骨架，但无音频采集能力） |
| EavesdropperHandler | `custom.handler.EavesdropperHandler` | 插嘴事件消费（已有骨架，但 LLM 调用返回硬编码文本） |

### 原有架构的问题

1. `EavesdropperEngine.handleSpeechDetected()` 是桩函数，**没有人调用它**
2. `EavesdropperHandler.callLLMWithSystemPrompt()` 返回硬编码 `"呵呵,你说得对!"`
3. **没有音频采集代码** — 系统只在唤醒词匹配后的 ASR 模式才采集音频
4. `EavesdropperEngine.stop()` 会永久杀死 scheduler，插嘴只能用一次

## 三、新增组件设计

### 3.1 VadAudioDetector — 音频采集与能量检测

**职责：** 独立的 `AudioRecord` 管理 + 能量检测状态机

**音频参数：**
- 采样率：16kHz
- 位深：16bit
- 声道：Mono
- 帧大小：20ms = 640 bytes

**能量算法**（复用 `VAD.java:1233-1259` 的 RMS 映射）：
```java
int energy = (int) ((10.0 * Math.log10((double)(2.0f * sum / (float)length + 1.0)) - 20.0) * 5.0);
// 映射到 0-100 范围
```

**状态机：**

```
        IDLE (空闲)
          │
          │ 能量 ≥ 阈值(30)
          ▼
    SPEECH_DETECTED (检测到语音, 累计计时)
          │
          │ 累计语音 ≥ 5000ms
          ▼
    SILENCE_WAIT (静音等待, 开始静音计时)
          │
          │ 静音 ≥ 2000ms
          ▼
    TRIGGERED (触发, 回调返回缓冲音频)

    回退路径:
    - SPEECH_DETECTED + 连续静音 ≥ 5000ms → IDLE (语音不足)
    - 任意状态 + cancel() → STOPPED
```

**循环缓冲区：**
- 最大 15 秒 PCM ≈ 480KB
- 回绕写入，防止内存溢出
- 触发时从写入位置回溯提取线性数据

**回调接口：**
```java
interface Listener {
    void onSpeechTriggered(byte[] audioBuffer, long speechDurationMs);
    void onStateChanged(State oldState, State newState);
}
```

### 3.2 EavesdropperASRRecognizer — 独立 ASR 会话

**职责：** 封装 `JniUscClient`，对预录制的音频块执行云端 ASR 识别

**生命周期：**
```
new JniUscClient()
  → a(server, port)          // createNative
  → a(ASR_SERVICE_MODE, NET) // 配置网络模式
  → g()                      // login
  → a("pcm")                 // 设置音频格式
  → a()                      // start
  → 循环 a(chunk, len)       // recognize 逐块送入
  → b()                      // stop
  → c()                      // getResult (JSON)
  → 解析 JSON 提取 text + score
  → e()                      // destroyNative
```

**音频块大小：** 3200 bytes ≈ 100ms

**结果解析：** 支持多种 JSON 格式：
- `net_nlu` 数组格式（云端识别）
- `local_asr` 数组格式（本地识别）
- 直接 `text` / `recognition_result` 字段

**线程安全：** `volatile boolean isRecognizing` 防止并发，通过 `ThreadUtils.execute()` 在后台线程执行。

### 3.3 EavesdropperSession — 会话编排器

**职责：** 串联 VadAudioDetector + EavesdropperASRRecognizer + ANTPipeline

**生命周期：**

```
startListening()
  → VadAudioDetector.startListening()  // 打开 AudioRecord
  → 能量检测触发
  → onSpeechTriggered(audioBuffer)
    → vadDetector.cancel()             // 停录音
    → asrRecognizer.recognize(buffer)  // 启动 ASR
      → onResult(text, confidence)
        → pipeline.fireUserEventTriggered(new EavesdropperTriggerEvent(text, confidence))
        → EavesdropperHandler 消费事件

cancel()
  → vadDetector.cancel()              // 立即停录音
  → asrRecognizer.cancel()            // 取消进行中的 ASR
```

## 四、修改组件说明

### 4.1 EavesdropperEngine.java — 重写

| 改动 | 说明 |
|------|------|
| 新增 `static EavesdropperEngine instance` | 全局单例，供 NLUDispatcher 静态调用 |
| 新增 `EavesdropperSession session` | 编排音频采集和 ASR 识别 |
| 新增 `static notifyWakeupDetected()` | 唤醒词检测时调用，立即取消插嘴会话 |
| 新增 `static notifyInteraction()` | 任意命令被处理时调用，取消插嘴会话 |
| 修改 `checkIdleThreshold()` | idle 达标时 `session.startListening()` |
| 修改 `checkListeningDuration()` | 超时调用 `session.cancel()`，**不杀 scheduler** |
| 修改 `stop()` | 只 cancel session，保持 scheduler 运行 |
| 新增 `destroy()` | 应用退出时完整清理 |
| 新增 `onInteractionDetected()` | 记录交互并重置 2 秒锁 |
| 删除 `handleSpeechDetected()` | 不再需要，session 直接发 pipeline 事件 |

### 4.2 NLUDispatcher.java — 通知钩子

| 位置 | 改动 |
|------|------|
| `onWakeupResult()` (第 346 行) | 取消注释并调用 `EavesdropperEngine.notifyWakeupDetected()` |
| `c(ANTHandlerContext, NLU)` (第 183 行) | 新增 `EavesdropperEngine.notifyInteraction()` |

**为什么在这两处：**
- `onWakeupResult`：唤醒词匹配时触发（小讯小讯、捣蛋鬼等），此时插嘴模式必须退出
- `c(ctx, nlu)`：所有 NLU 命令分发入口（音量、播放、天气等），覆盖所有命令处理

### 4.3 EavesdropperHandler.java — 接入大模型

| 改动 | 说明 |
|------|------|
| 新增 `OpenAIClient openAIClient` | 大模型客户端 |
| 新增 `onASREventEngineInitDone()` | 初始化 AIConfig + OpenAIClient |
| 替换 `callLLMWithSystemPrompt()` | 桩函数 `"呵呵,你说得对!"` → `openAIClient.chat(userInput, systemPrompt)` |
| `generateAndPlayEavesdropResponse()` | 新增 `ctx.stopWakeup()` + `ctx.stopASR()` (与 PhicommChatHandler 一致) |

### 4.4 ExampleANTEngineInitializer.java — 注册 Handler

```java
pipeline.addLast(new PhicommChatHandler());
// 插嘴 Handler: 消费 EavesdropperTriggerEvent, 调用大模型回复
pipeline.addLast(new EavesdropperHandler());
pipeline.addLast(new DefaultChatHandler());
```

### 4.5 PhicommInitializeHandler.java — 传递 ANTEngine 引用

```java
// 修改前
mEavesdropperEngine = new EavesdropperEngine(this.mContext, pipeline);

// 修改后
mEavesdropperEngine = new EavesdropperEngine(this.mContext, pipeline, this.mANTEngine);
```

`ANTEngine` 引用用于 ASR 识别器获取用户 ID、app key 等配置。

## 五、完整数据流

```
┌─────────────────────────────────────────────────────────────────┐
│                      空闲 30 秒+                                 │
└─────────────────────────┬───────────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│  EavesdropperEngine.checkIdleThreshold()                         │
│    → PersonaManager.shouldTriggerActiveListening() = true        │
│    → session.startListening()                                    │
└─────────────────────────┬───────────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│  VadAudioDetector: AudioRecord(16kHz/16bit/mono)                  │
│    → 每 20ms 帧计算能量值 (RMS→dB→0-100)                          │
│    → 状态机: IDLE → SPEECH_DETECTED → SILENCE_WAIT → TRIGGERED   │
│    → 5秒累计语音 + 2秒静音截断                                    │
└─────────────────────────┬───────────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│  EavesdropperSession.startASRRecognition(audioBuffer)            │
│    → vadDetector.cancel()    // 停录音                           │
│    → asrRecognizer.recognize(buffer)  // 独立 ASR 会话            │
└─────────────────────────┬───────────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│  EavesdropperASRRecognizer                                        │
│    → JniUscClient 流式识别 (3200 bytes/chunk)                     │
│    → stop() → getResult() → 解析 JSON → text + confidence        │
└─────────────────────────┬───────────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│  pipeline.fireUserEventTriggered(EavesdropperTriggerEvent)       │
└─────────────────────────┬───────────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│  EavesdropperHandler.eventReceived()                             │
│    → 激活 EAVESDROPPER 人格 ("贫嘴伙伴")                          │
│    → OpenAIClient.chat(userInput, systemPrompt)                  │
│    → ctx.playTTS(response)                                       │
│    → PersonaManager.recordInteraction()  // 重置 idle 计时       │
└─────────────────────────┬───────────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│  EavesdropperEngine scheduler 继续运行                            │
│    → 下次空闲 30 秒后重复整个流程                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 退出路径

```
任意时刻:
├─ 唤醒词匹配 → EavesdropperEngine.notifyWakeupDetected()
│               → session.cancel()
│               → 2秒锁防止重复触发
│
├─ NLU 命令处理 → EavesdropperEngine.notifyInteraction()
│                 → session.cancel()
│                 → 2秒锁
│
├─ 语音不足 5 秒 → VadAudioDetector 自动回到 IDLE
│
└─ 监听超时 (5分钟) → EavesdropperEngine.checkListeningDuration()
                      → session.cancel()
```

## 六、配置参数

| 参数 | 值 | 文件/位置 | 说明 |
|------|-----|-----------|------|
| 空闲触发阈值 | 30 秒 | PersonaConfig.EAVESDROPPER.idleTriggerThresholdMs | 空闲多久后才启动插嘴监听 |
| 最大监听时长 | 5 分钟 | PersonaConfig.EAVESDROPPER.maxListeningDurationMs | 单次插嘴会话最长持续时间 |
| 累计语音触发 | 5000ms | VadAudioDetector.CUMULATIVE_SPEECH_MS | 需要累计多少秒语音才进入静音等待 |
| 静音截断 | 2000ms | VadAudioDetector.SILENCE_TRIGGER_MS | 语音达标后多少秒静音触发 ASR |
| 静音重置 | 5000ms | VadAudioDetector.CONTINUOUS_SILENCE_RESET_MS | 语音不足时连续静音多久回到 IDLE |
| 能量阈值 | 30 (0-100) | VadAudioDetector.ENERGY_THRESHOLD | RMS 能量超过此值认为有人声 |
| 最大缓冲 | 15 秒 | VadAudioDetector.MAX_BUFFER_SECONDS | 循环缓冲区上限 ~480KB |
| ASR 块大小 | 3200 bytes | EavesdropperASRRecognizer.CHUNK_SIZE | 约 100ms 音频 |
| 插嘴响应延迟 | 1500ms | PersonaConfig.EAVESDROPPER.eavesdropResponseDelayMs | 模拟思考时间 |
| 唤醒词处理锁 | 2000ms | EavesdropperEngine | 唤醒词匹配后 2 秒内不启动新会话 |

## 八、验证方式

### 日志关键词

```bash
logcat | grep -E "VadAudioDetector|EavesdropperASR|EavesdropperSession|EavesdropperHandler"
```

### 预期日志序列

| 阶段 | 预期日志 |
|------|---------|
| 空闲触发 | `Eavesdropper session started` |
| 人声检测 | `VAD state: IDLE -> SPEECH_DETECTED` |
| 语音达标 | `VAD state: SPEECH_DETECTED -> SILENCE_WAIT` |
| 静音触发 | `VAD state: SILENCE_WAIT -> TRIGGERED` |
| ASR 开始 | `Streaming XXX bytes to ASR` |
| ASR 结果 | `ASR result: "xxx", confidence=X` |
| 大模型调用 | `Eavesdrop response: xxx` |
| 唤醒词退出 | `Wakeup detected, cancelling eavesdropper session` |
| 命令退出 | `Interaction detected, cancelling eavesdropper session` |

### 测试场景

| 场景 | 预期结果 |
|------|---------|
| 空闲 30 秒后说话 5 秒+ | 触发 ASR → 大模型回复 → TTS |
| 说话不足 5 秒就停止 | 不触发，回到 IDLE 继续监听 |
| 说话过程中说唤醒词 | 插嘴模式退出，进入唤醒词处理流程 |
| 说话过程中下音量命令 | 插嘴模式退出，音量命令生效 |
| 插嘴回复后空闲 30 秒 | 再次自动进入插嘴监听 |
| 说"给我闭嘴" | 闭嘴 1 小时，插嘴模式不再触发 |
