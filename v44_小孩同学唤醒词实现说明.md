## 二、实现原理

### 2.1 唤醒词切换机制(已验证)

根据之前的技术分析,R1 设备使用 **云知声 USC 引擎** 的 **JSGF 词槽机制**:

```
用户输入新唤醒词 "小孩同学"
    ↓
调用 NativeANTEngine.setWakeupWord0(["小孩同学"], false)
    ↓
├─ 停止当前ASR识别
├─ 取消引擎当前操作
├─ 更新词槽数据: this.j.b(hashMap)
│    └─ USC引擎本地编译JSGF (compile code = 0, 毫秒级)
├─ 保存到SharedPreferences
└─ 触发引擎重新加载: this.j.a(1018, ...)
    ↓
回调: onWakeupEventSetWakeUpWordDone()
    ↓
新唤醒词立即生效!
```

**关键特性**:
- ✅ **纯本地操作**,不需要云端训练
- ✅ **即时生效** (<1秒)
- ✅ **支持多个唤醒词**同时生效
- ✅ **支持任意2-6个中文字**

### 2.2 代码实现位置

在 `PhicommInitializeHandler.java` 中添加初始化逻辑:

**文件**: `/Users/fang/Desktop/R1_DEVICE-main/app/src/main/java/com/phicomm/speaker/device/custom/handler/PhicommInitializeHandler.java`

**关键代码**:

```java
/**
 * 设置自定义唤醒词 "小孩同学"
 * 原理: 通过 JSGF 词槽本地编译,无需云端训练,即时生效
 */
private void initCustomWakeupWord() {
    String customWakeupWord = "小孩同学";
    
    // 检查是否已经设置过,避免重复设置
    List<String> currentWords = UserPerferenceUtil.getMainWakeupWord(this.mContext);
    if (currentWords != null && currentWords.contains(customWakeupWord)) {
        Log.d(TAG, "Custom wakeup word already set: " + customWakeupWord);
        return;
    }
    
    Log.d(TAG, "Setting custom wakeup word: " + customWakeupWord);
    
    // 构建新的唤醒词列表: 保留原有默认唤醒词 + 添加自定义唤醒词
    List<String> newWakeupWords = new ArrayList<>();
    newWakeupWords.add(customWakeupWord);
    // 保留原有唤醒词(如"小讯小讯")
    newWakeupWords.addAll(UserPerferenceUtil.getDefaultWakeupWord(this.mContext));
    
    try {
        // 1. 保存到 SharedPreferences
        UserPerferenceUtil.setWakeupWord(this.mContext, newWakeupWords);
        
        // 2. 调用引擎更新唤醒词(会触发本地 JSGF 编译)
        this.mANTEngine.stopWakeup();
        this.mANTEngine.updateWakeupWord(newWakeupWords);
        
        // 3. 触发唤醒词变更事件
        this.mANTEngine.pipeline().fireUserEventTriggered(new ChangeWakeupWordEvent(customWakeupWord));
        
        Log.d(TAG, "Custom wakeup word set successfully: " + newWakeupWords);
    } catch (Exception e) {
        Log.e(TAG, "Failed to set custom wakeup word: " + customWakeupWord, e);
    }
}
```

**调用时机**: 在 `initPhicommBusiness()` 中调用,确保设备启动时自动设置。

### 5.1 唤醒词验证规则

在 `CustomApiManager.isWakeupWordValid()` 中定义:

```java
private boolean isWakeupWordValid(@NonNull String wakeupWord) {
    // 长度必须在 4-6 个字符之间
    if (!TextUtils.isEmpty(wakeupWord) && wakeupWord.length() >= 4 && wakeupWord.length() <= 6) {
        return true;
    }
    return false;
}
```

**"小孩同学" = 4 个字符**,符合要求 ✅

### 5.2 多唤醒词共存机制

R1 设备支持**多个唤醒词同时生效**,存储在 SharedPreferences 中:

```java
// UserPerferenceUtil.setWakeupWord()
public static void setWakeupWord(Context context, List<String> wakeupWord) {
    HashSet hashSet = new HashSet();
    hashSet.addAll(wakeupWord);
    SharedPreferencesHelper.getInstance(context, SP_NAME)
        .saveStringSetValue(SP_WAKEUP_WORD, hashSet);
}

// 获取时合并默认唤醒词
public static List<String> getMainWakeupWord(Context context) {
    HashSet hashSet = new HashSet();
    Set<String> stringSetValue = SharedPreferencesHelper.getInstance(context, SP_NAME)
        .getStringSetValue(SP_WAKEUP_WORD);
    if (stringSetValue != null && stringSetValue.size() > 0) {
        hashSet.addAll(stringSetValue);
    }
    hashSet.addAll(getDefaultWakeupWord(context)); // 合并默认词
    ArrayList arrayList = new ArrayList();
    arrayList.addAll(hashSet);
    return arrayList;
}
```

## 六、注意事项

### 6.1 重启后是否保留?

**✅ 会保留**。唤醒词存储在 SharedPreferences 中,重启后自动加载。

### 6.2 如何恢复默认唤醒词?

如果需要移除"小孩同学",可以:

1. **方法 A**: 通过局域网 API 调用 `MODIFY_WAKE_UP_WORD`,传入原有唤醒词
2. **方法 B**: 清除 SharedPreferences 中的 `SP_WAKEUP_WORD` 字段
3. **方法 C**: 重新刷机/恢复出厂设置

### 6.3 是否可以添加更多唤醒词?

**✅ 可以**。只需修改 `initCustomWakeupWord()` 方法:

```java
List<String> newWakeupWords = new ArrayList<>();
newWakeupWords.add("小孩同学");
newWakeupWords.add("天猫精灵");  // 添加更多
newWakeupWords.addAll(UserPerferenceUtil.getDefaultWakeupWord(this.mContext));
```

**注意**: 唤醒词数量不宜过多,可能影响识别准确率。
