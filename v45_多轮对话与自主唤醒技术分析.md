# R1 设备多轮对话与代码自主唤醒技术分析

> **分析日期**: 2026-05-13  
> **设备**: 斐讯 R1 (Android 5.1)  
> **代码源**: `/Users/fang/Desktop/R1_DEVICE-main`  
> **目的**: 分析是否可以实现"成语接龙"等多轮对话场景,AI 说完后自动进入聆听状态,无需每次都说唤醒词

---

## 一、核心结论

### ✅ **可以实现代码自主唤醒,支持多轮对话!**

ANT 引擎提供了 `ctx.enterASR()` 接口,可以让设备在 TTS 播放完成后**自动进入语音识别状态**,无需唤醒词即可继续聆听。

## 二、唤醒机制完整解析

### 2.1 引擎状态机

```
┌─────────────────────────────────────────────────────────────┐
│                     ANT 引擎状态机                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [Idle 空闲] ──enterWakeup()──→ [Wakeup 唤醒监听]            │
│      ↑                              ↓                       │
│      │                          检测到唤醒词                   │
│      │                              ↓                       │
│      │                        [ASR 语音识别]                  │
│      │                              ↓                       │
│      │                          识别完成                      │
│      │                              ↓                       │
│      │                         [NLU 语义理解]                 │
│      │                              ↓                       │
│      │                       [Handler 处理意图]               │
│      │                              ↓                       │
│      │                          [TTS 播报]                   │
│      │                              ↓                       │
│      │                      onTTSEventPlayingEnd()           │
│      │                              ↓                       │
│      └────enterWakeup(false)←── [exit() 退出会话]             │
│                                                             │
│  可选路径(多轮对话):                                          │
│      [TTS 播报] → onTTSEventPlayingEnd()                    │
│                      ↓                                      │
│                ctx.enterASR() ←── 直接进入识别!               │
│                      ↓                                      │
│                [无需唤醒词,继续聆听]                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 关键 API 说明

| API | 功能 | 说明 |
|-----|------|------|
| `ctx.enterWakeup(false)` | 进入唤醒监听状态 | 需要说唤醒词才能触发识别 |
| `ctx.enterASR()` | **直接进入语音识别** | **无需唤醒词,立即开始聆听** ✅ |
| `ctx.cancelEngine()` | 取消整个引擎 | 会回到 Idle 状态 |
| `ctx.cancelTTS()` | 取消 TTS 播报 | 停止当前语音播放 |
| `ctx.cancelASR()` | 取消当前识别 | 停止聆听 |

### 3.1 已有的多轮对话实现:AbstractMemoHandler

**文件**: `app/src/main/java/com/unisound/vui/handler/session/memo/AbstractMemoHandler.java`

这是**备忘录功能**的 Handler,已经实现了多轮对话!

```java
// 第 162-176 行
@Override
public boolean onTTSEventPlayingEnd(ANTHandlerContext ctx) {
    if (this.eventReceived) {
        reset();
        // 关键判断: 是否需要补充信息?
        if (!ctx.hasAttr(NEED_SUPPLEMENT) || 
            !((Boolean) ctx.attr(NEED_SUPPLEMENT).getAndRemove()).booleanValue()) {
            // 不需要补充 → 回到唤醒状态
            recoveryHandlerPriority();
            ctx.enterWakeup(false);  // ← 需要唤醒词
            ctx.fireUserEventTriggered(ExoConstants.DO_RESUME);
        } else {
            // 需要补充 → 直接进入 ASR! ✅
            ctx.enterASR();  // ← 无需唤醒词,自动聆听
        }
    }
    return super.onTTSEventPlayingEnd(ctx);
}

// 第 188-196 行: 设置 NEED_SUPPLEMENT 标志
public String dealWithAnswer(ANTHandlerContext ctx, General answerGeneral) {
    if (answerGeneral == null) {
        return null;
    }
    // 标记需要补充信息
    ctx.attr(NEED_SUPPLEMENT).set(true);  // ← 设置标志
    setPriority(PriorityMap.PRIORITY_MAX); // 提升优先级,抢占 Pipeline
    return answerGeneral.getText();
}
```

## 四、实现"成语接龙"多轮对话的方案

### 4.1 核心思路

修改 `PhicommChatHandler` 的 `onTTSEventPlayingEnd()` 方法:
- TTS 播放完成后,**不调用 `exit()`**
- 改为调用 `ctx.enterASR()` 直接进入下一轮识别
- 使用一个标志位控制是否继续多轮对话

### 4.2 代码实现

**文件**: `app/src/main/java/com/phicomm/speaker/device/custom/handler/PhicommChatHandler.java`

```java
public class PhicommChatHandler extends SimpleUserEventInboundHandler {
    private static final String TAG = "PhicommChatHandler";
    
    // 新增: 多轮对话标志
    private boolean isMultiTurnMode = false;
    private int multiTurnCount = 0;
    private static final int MAX_MULTI_TURN_COUNT = 50;  // 最多 50 轮
    
    @Override
    public boolean acceptInboundEvent0(NLU evt) {
        if (evt == null) return false;
        String service = evt.getService();
        if (!"cn.yunzhisheng.chat".equals(service)) return false;
        
        String text = evt.getText();
        if (text != null && !text.isEmpty()) {
            // 检查是否是退出指令
            if (text.contains("退出") || text.contains("结束") || text.contains("停止")) {
                LogMgr.d(TAG, "用户请求退出多轮对话");
                isMultiTurnMode = false;
                // 放行给 DefaultChatHandler 处理退出逻辑
                return false;
            }
            
            // 检查是否是开始游戏指令
            if (text.contains("成语接龙") || text.contains("开始游戏")) {
                LogMgr.d(TAG, "用户请求开始成语接龙");
                isMultiTurnMode = true;
                multiTurnCount = 0;
            }
            
            // 音乐/天气关键词放行(保持原有逻辑)
            if (containsAny(text, MUSIC_KEYWORDS)) {
                LogMgr.d(TAG, "放行音乐意图: " + text);
                isMultiTurnMode = false;  // 退出多轮模式
                return false;
            }
            if (containsAny(text, WEATHER_KEYWORDS)) {
                LogMgr.d(TAG, "放行天气意图: " + text);
                isMultiTurnMode = false;  // 退出多轮模式
                return false;
            }
        }
        return true;  // 消费,走大模型
    }
    
    @Override
    protected void eventReceived(NLU evt, ANTHandlerContext ctx) throws Exception {
        super.eventReceived(evt, ctx);
        this.ctx = ctx;
        this.eventReceived = true;
        
        String userInput = evt.getText();
        LogMgr.d(TAG, "eventReceived: userInput=" + userInput + 
                      ", multiTurnMode=" + isMultiTurnMode + 
                      ", count=" + multiTurnCount);
        
        ThreadUtils.execute(() -> {
            try {
                // 调用大模型
                String reply = callLLM(userInput);
                
                // 如果是多轮模式,增加计数
                if (isMultiTurnMode) {
                    multiTurnCount++;
                    LogMgr.d(TAG, "Multi-turn count: " + multiTurnCount);
                    
                    // 检查是否超过最大轮数
                    if (multiTurnCount >= MAX_MULTI_TURN_COUNT) {
                        LogMgr.d(TAG, "达到最大轮数限制,自动退出");
                        isMultiTurnMode = false;
                        ctx.playTTS("成语接龙游戏结束,您真厉害!共完成了 " + 
                                   multiTurnCount + " 轮对话。");
                        return;
                    }
                }
                
                // 播放 TTS
                ctx.playTTS(reply);
                
            } catch (Exception e) {
                LogMgr.e(TAG, "LLM call failed", e);
                ctx.playTTS("抱歉,我暂时无法回答这个问题。");
            }
        });
    }
    
    /**
     * TTS 播放完成后的回调
     * 关键: 根据多轮模式决定下一步动作
     */
    @Override
    public boolean onTTSEventPlayingEnd(ANTHandlerContext ctx) {
        if (!this.eventReceived) {
            return super.onTTSEventPlayingEnd(ctx);
        }
        
        LogMgr.d(TAG, "onTTSEventPlayingEnd: multiTurnMode=" + isMultiTurnMode);
        
        if (isMultiTurnMode) {
            // 多轮模式: 直接进入 ASR,无需唤醒词! ✅
            LogMgr.d(TAG, "Multi-turn mode: entering ASR directly");
            reset();  // 重置会话状态
            
            // 进入语音识别(无需唤醒词)
            ctx.enterASR();
            
            // 可选: 播放提示音(类似唤醒成功的"滴滴"声)
            // ctx.playTTS("叮");  // 或者使用系统提示音
            
            return true;  // 消费事件
        } else {
            // 普通模式: 正常退出,回到唤醒状态
            LogMgr.d(TAG, "Normal mode: exiting to wakeup");
            exit();
            return true;
        }
    }
    
    @Override
    public void doInterrupt(ANTHandlerContext ctx, String interruptType) {
        if (this.eventReceived) {
            ctx.cancelTTS();
            // 中断时也要退出多轮模式
            isMultiTurnMode = false;
            multiTurnCount = 0;
            
            if (!ExoConstants.DO_ONE_SHOT_INTERRUPT.equals(interruptType)) {
                ctx.enterWakeup(false);
            }
            reset();
        }
    }
    
    private void exit() {
        try {
            this.ctx.cancelTTS();
            // 注意:不要调用 cancelEngine(),它会取消整个 ANT 引擎
        } catch (Throwable ignored) {}
        reset();
        // 重置多轮标志
        isMultiTurnMode = false;
        multiTurnCount = 0;
    }
    
    private String callLLM(String userInput) throws Exception {
        AIConfig config = AIConfig.getInstance();
        OpenAIClient client = new OpenAIClient(config);
        return client.chat(userInput);
    }
    
    // ... 其他方法保持不变 ...
}
```

private String callLLMWithContext(List<String> history) throws Exception {
    AIConfig config = AIConfig.getInstance();
    
    // 构建带上下文的请求
    StringBuilder prompt = new StringBuilder();
    prompt.append("当前正在进行成语接龙游戏。规则:\n");
    prompt.append("1. 每个成语的最后一个字作为下一个成语的第一个字\n");
    prompt.append("2. 不能重复使用已说过的成语\n");
    prompt.append("3. 必须是四字成语\n\n");
    prompt.append("对话历史:\n");

    prompt.append("\n请回复下一个成语(只回复成语本身,不要其他内容):");
    
    // 调用大模型
    OpenAIClient client = new OpenAIClient(config);
    return client.chat(prompt.toString());
}

## 六、注意事项

### 6.1 麦克风状态

调用 `ctx.enterASR()` 后,麦克风会**立即开启**,设备开始录音。此时:
- ✅ 用户可以直接说话
- ⚠️ 如果 TTS 还没完全停止,可能会录到 TTS 的尾音
- **解决方法**: 在 `enterASR()` 前加一个小延迟

```java
@Override
public boolean onTTSEventPlayingEnd(ANTHandlerContext ctx) {
    if (isMultiTurnMode) {
        reset();
        
        // 延迟 500ms 再进入 ASR,确保 TTS 完全停止
        new Handler(Looper.getMainLooper()).postDelayed(() -> {
            ctx.enterASR();
        }, 500);
        
        return true;
    }
    // ...
}
```

### 6.2 优先级管理

在多轮模式下,需要保持 Handler 的高优先级,避免被其他 Handler 拦截:

```java
@Override
public boolean onASREventEngineInitDone(ANTHandlerContext ctx) {
    this.ctx = ctx;
    // 在多轮模式下,提升优先级到最高
    if (isMultiTurnMode) {
        setPriority(PriorityMap.PRIORITY_MAX);
    }
    return true;
}
```

### 6.3 异常处理

网络错误、大模型超时等异常情况下,也要退出多轮模式:

```java
ThreadUtils.execute(() -> {
    try {
        String reply = callLLM(userInput);
        ctx.playTTS(reply);
    } catch (Exception e) {
        LogMgr.e(TAG, "LLM call failed", e);
        isMultiTurnMode = false;
        ctx.playTTS("抱歉,网络异常,游戏结束");
        ctx.enterWakeup(false);
    }
});
```

## 八、总结

### ✅ 可行性确认

1. **技术可行性**: ANT 引擎原生支持 `ctx.enterASR()`,可以直接进入识别状态,无需唤醒词
2. **代码证据**: AbstractMemoHandler、PhicommAlarmHandler 等已经实现了多轮对话
3. **实现难度**: 低,只需修改 PhicommChatHandler 的 `onTTSEventPlayingEnd()` 方法

### 🎯 核心改造点

```java
// 改造前(单轮):
@Override
public boolean onTTSEventPlayingEnd(ANTHandlerContext ctx) {
    exit();  // ← 调用 exit() → enterWakeup() → 需要唤醒词
    return true;
}

// 改造后(多轮):
@Override
public boolean onTTSEventPlayingEnd(ANTHandlerContext ctx) {
    if (isMultiTurnMode) {
        reset();
        ctx.enterASR();  // ← 直接进入识别! 无需唤醒词! ✅
        return true;
    } else {
        exit();
        return true;
    }
}
```
