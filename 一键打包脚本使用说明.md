# R1_DEVICE 一键打包脚本使用说明

## 📦 快速开始

修改完代码后，只需一条命令即可完成全量编译、打包、安装：

```bash
cd /Users/fang/Desktop/R1_DEVICE-main
./build_and_deploy.sh <版本号>
```

### 示例

```bash
# 打包 v62 版本
./build_and_deploy.sh 62

# 打包 v63 版本
./build_and_deploy.sh 63
```

## 🎯 脚本功能

脚本会自动完成以下 9 个步骤：

1. ✅ **清理并编译** - `gradlew clean :app:assembleRelease`
2. ✅ **提取类文件** - 提取所有 custom 目录下的 .class 文件
3. ✅ **编译 DEX** - 使用 dx 工具编译成 classes.dex
4. ✅ **转换 smali** - 使用 baksmali 将 DEX 转为 smali
5. ✅ **注入 smali** - 将 smali 注入到 original_v2 目录
6. ✅ **检查注册** - 验证关键 Handler 是否都已注册
7. ✅ **编译最终 DEX** - 使用 smali.jar 编译完整 DEX
8. ✅ **构造 APK** - 使用 Janus 漏洞构造 APK
9. ✅ **安装到设备** - 推送、安装、清缓存、重启

## 🛠️ 故障排查

### 问题1：编译失败

**错误信息**：`BUILD FAILED`

**解决方案**：
```bash
# 检查 Java 环境
echo $JAVA_HOME
java -version

# 手动清理 Gradle 缓存
./gradlew clean
rm -rf ~/.gradle/caches/
```

### 问题2：ADB 连接失败

**错误信息**：`device not found` 或 `connection refused`

**解决方案**：
```bash
# 检查设备连接
adb devices

# 如果是 WiFi 连接，重新连接
adb connect 192.168.5.9:5555

# 如果是 USB 连接
adb kill-server
adb start-server
adb devices
```

### 问题3：安装失败

**错误信息**：`INSTALL_FAILED_UPDATE_INCOMPATIBLE`

**解决方案**：
```bash
# 卸载旧版本
adb shell pm uninstall com.phicomm.speaker.device

# 重新运行脚本
./build_and_deploy.sh 62
```

### 问题4：APK 推送太慢

**原因**：WiFi 速度慢

**解决方案**：
```bash
# 使用 USB 连接（更快）
adb usb
adb push janus_unisound_v62.apk /sdcard/
```
